## 使用CNN模型分類笑容
相較於Fashion-MNIST的圖檔，Celeba的圖檔較大(28 x 28 x 1 vs 218 x 178 x 3)，但所需分類的類別較少(10 vs 2)。之前我們達到了88%的正確率，此練習有二目的：

1. 測試是否相同的模型可以分類不同類別的圖檔，
2. 測試同樣的模型對資料量較大的圖檔的表現。

之前我們使用了一個非常簡單的CNN模型，如下所示。

```

model = keras.models.Sequential(
        [
            keras.layers.Conv2D(
                filters=3,
                kernel_size=7,
                activation="relu",
                padding="SAME",
                input_shape=[28, 28, 1],
            ),
            keras.layers.MaxPooling2D(2),
            keras.layers.Flatten(),
            keras.layers.Dense(units=10, activation="softmax"),
        ]
    )

```

為了進行二元分類作業，我們需將最後的輸出層改為，`keras.layers.Dense(units=1, activation='sigmoid')`，使用sigmoid函式；並將輸入層中，告知模型的樣本各個維度的長度。

```
    model = keras.models.Sequential(
        [
            keras.layers.Conv2D(
                filters=3,
                kernel_size=7,
                activation="relu",
                padding="SAME",
                input_shape=[218, 178, 3],
            ),
            keras.layers.MaxPooling2D(2),
            keras.layers.Flatten(),
            keras.layers.Dense(units=1, activation='sigmoid')
        ]
    )

```

因為此資料集較大，一次將所有的圖檔資料傳送進GPU記憶體，會導致記憶體超載。因此我們僅使用前3000張圖檔。
* 另一解決方案是使用檔名來做Shuffle，在model fitting之時以批次的方式，僅將每一批次的檔案送進GPU記憶體中。
